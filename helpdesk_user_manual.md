# คู่มือการใช้งานระบบ IT Helpdesk Support System

**ระบบ IT Helpdesk** ถูกออกแบบมาเพื่ออำนวยความสะดวกในการรับแจ้งปัญหา (Ticket) การติดตามสถานะ และการแก้ปัญหาอย่างเป็นระบบ โดยแบ่งผู้ใช้งานออกเป็น 3 ระดับหลัก ได้แก่:
1. **End User (ผู้ใช้งานทั่วไป)** - ผู้แจ้งปัญหา
2. **Tier 1 Support (พนักงาน Helpdesk)** - ผู้รับเรื่อง ประเมินเบื้องต้น และแจกจ่ายงาน
3. **Tier 2 Support (ช่างเทคนิค/ผู้เชี่ยวชาญ)** - ผู้แก้ไขปัญหาเชิงลึกและสรุปปิดงาน

---

## 1. สำหรับผู้ใช้งานทั่วไป (End User)

### 1.1 การสร้างรายการแจ้งปัญหาใหม่ (Create Ticket)
ผู้ใช้งานสามารถแจ้งปัญหาที่พบได้ผ่านหน้าฟอร์มแจ้งปัญหา 

1. ไปที่เมนู **"แจ้งปัญหาใหม่"** ที่แถบเมนูด้านซ้าย
2. **ข้อมูลผู้แจ้ง:** 
   - ให้เลือก **Business Unit (BU)** ต้นสังกัดของคุณเป็นอันดับแรก
   - จากนั้นเลือก **ผู้แจ้ง (Reporter)** (ระบบจะแสดงเฉพาะชื่อที่อยู่ใน BU ที่เลือก)
   - เลือก **จุดเกิดเหตุ (Location)** ว่าปัญหาเกิดที่สาขาหรืออาคารใด
3. **รายละเอียดปัญหา:**
   - กรอก **หัวข้อ (Subject)** แบบสั้นๆ เข้าใจง่าย (เช่น POS สแกนบาร์โค้ดไม่ได้)
   - เลือก **ประเภท (Software / Hardware)** และ **ระบบที่มีปัญหา (System)**
   - เลือกระดับ **ความเร่งด่วน (Priority)**
   - อธิบาย **อาการที่พบ (Symptom)** และ **รายละเอียดเพิ่มเติม**
4. **แนบไฟล์:** สามารถลากไฟล์รูปภาพหน้าจอ Error หรือเอกสารอ้างอิงมาวางในกล่องได้
5. กดปุ่ม **"ส่งแจ้งปัญหา"** ระบบจะสร้างหมายเลข Ticket ให้ทันที (เช่น INC-2026-001)

*(คำแนะนำ: ผู้ใช้งานสามารถติดตามสถานะการแก้ไขปัญหาได้จากหน้า All Tickets)*

## 2. สำหรับเจ้าหน้าที่รับเรื่อง (Tier 1 Support)

Tier 1 คือด่านหน้าของระบบ มีหน้าที่ตรวจสอบความถูกต้อง ให้การช่วยเหลือเบื้องต้น และกรองงานก่อนส่งต่อ

### 2.1 หน้า Dashboard (รับเรื่อง / ประเมิน)
1. ไปที่เมนู **"รับเรื่อง / ประเมิน"** ภายใต้หมวด TIER 1 SUPPORT
2. หน้าจอจะแสดงรายการ Ticket ทั้งหมดที่เพิ่งถูกแจ้งเข้ามา (สถานะเป็น `New` สีแดง) และงานที่กำลังประเมินอยู่ (`Assessing` สีน้ำเงิน)
3. หากมี Ticket ใหม่ ให้คลิกปุ่ม **"รับเรื่อง"** ที่ท้ายตารางของงานนั้นๆ

`[แทรกรูปภาพ: หน้าจอ Tier 1 Dashboard และปุ่มรับเรื่อง]`

### 2.2 การประเมินและโอนงาน (Assessment & Escalation)
เมื่อเข้าสู่หน้ารายละเอียดของ Ticket แล้ว Tier 1 จะต้องทำหน้าที่:
1. **อ่านรายละเอียด:** ตรวจสอบอาการและสิ่งที่ User แจ้งมา
2. **บันทึกผลการประเมินเบื้องต้น:**
   - กรอก **ผลการประเมิน** และ **สาเหตุที่คาดว่าจะเป็น**
   - ระบุ **สิ่งที่ได้ดำเนินการไปแล้ว (Action Taken)** เช่น แนะนำให้ User ลองรีสตาร์ทเครื่องแล้วแต่ไม่หาย
   - กด **"บันทึกผลการประเมิน"**
3. **การส่งต่อให้ Tier 2 (Escalate):** 
   - หากปัญหาต้องใช้ช่างผู้เชี่ยวชาญแก้ไข ให้เลื่อนลงมาที่หัวข้อ "ส่งต่อให้ Tier 2"
   - กรอก **เหตุผลที่ต้องส่งต่อ** (เช่น ต้องแก้ไขข้อมูลใน Database)
   - กดปุ่ม **"โอนงานให้ Tier 2"** (สถานะ Ticket จะเปลี่ยนเป็น `Escalated` และวิ่งไปหา Tier 2 ทันที)

`[แทรกรูปภาพ: หน้าจอ Tier 1 Ticket Detail และฟอร์มการโอนงาน]`

## 3. สำหรับช่างเทคนิค (Tier 2 Support)

Tier 2 มีหน้าที่แก้ไขปัญหาเชิงลึกที่ได้รับการโอนงานมาจาก Tier 1 และทำการสรุปปิดงาน

### 3.1 หน้า Dashboard (แก้ปัญหา)
1. ไปที่เมนู **"แก้ปัญหา"** ภายใต้หมวด TIER 2 SUPPORT
2. ตารางจะแสดงรายการงานที่ถูกส่งต่อมา (สถานะ `Escalated` สีส้ม) และงานที่กำลังแก้ไข
3. คลิกปุ่ม **"เปิดดู"** เพื่อเข้าไปดูรายละเอียดและประวัติที่ Tier 1 ทำมาให้

`[แทรกรูปภาพ: หน้าจอ Tier 2 Dashboard]`

### 3.2 การบันทึกแก้ไขและการปิดงาน (Resolution)
1. เมื่อดำเนินการซ่อมแซมหรือแก้ไขระบบจนเสร็จสิ้นแล้ว ให้ช่างเทคนิคบันทึกข้อมูลการแก้ไข
2. ในหน้ารายละเอียด Ticket กรอกข้อมูล:
   - **สาเหตุของปัญหา (Root Cause):** สาเหตุที่แท้จริงที่ทำให้เกิด Error
   - **วิธีแก้ไข (Resolution Details):** วิธีการและขั้นตอนที่ช่างทำลงไปเพื่อแก้ปัญหา
3. กดปุ่ม **"บันทึกและเปลี่ยนสถานะเป็น RESOLVED"**
4. Ticket จะถูกตั้งเป็นสถานะแก้ไขเสร็จสิ้น รอให้ End User เข้ามากดยืนยันการปิดงาน

`[แทรกรูปภาพ: หน้าจอ Tier 2 Ticket Detail พร้อมช่องกรอก Root Cause และ Resolution]`

## 4. การจัดการข้อมูลตั้งต้น (Master Data)

เมนูในส่วนนี้สำหรับ Admin เพื่อจัดการโครงสร้างองค์กร ซึ่งจะส่งผลต่อแบบฟอร์มการแจ้งปัญหาทั้งหมด

1. **Business Units (BU):** จัดการรายชื่อบริษัทหรือแผนกต่างๆ
2. **Systems / Software:** จัดการรายชื่อระบบงาน (เช่น ระบบ POS, ระบบบัญชี) และกำหนดให้ได้ว่าใครเป็น Owner
3. **Locations:** จัดการสถานที่ตั้งหรือสาขา โดยสามารถ **ผูกกับ BU (Business Unit)** ได้ เพื่อให้ตอนแจ้งปัญหา ระบบสามารถกรองสถานที่ตามบริษัทที่พนักงานสังกัดอยู่ได้อัตโนมัติ
4. **Users:** จัดการข้อมูลพนักงานและกำหนดสิทธิ (Role) ว่าเป็น User ทั่วไป, Tier 1, หรือ Tier 2
